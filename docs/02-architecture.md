## Docker コンテナの実際の構成

```
ホストマシン（あなたのPC or サーバー）
│
├── /path/to/nanoclaw/          ← nanoclaw のソースコード
│       ├── src/
│       ├── groups/             ← グループ設定
│       └── data/               ← データ
│
└── Docker コンテナ（nanoclaw が動いている）
        │
        ├── /workspace/project/ ← ホストのソースを読み取り専用でマウント
        ├── /workspace/group/   ← グループデータ（書き込み可能・永続）
        │       └── nanoclaw-learning/  ← このリポジトリ！
        └── /home/node/         ← 一時領域（セッション終了で消える）
```

## 各レイヤーの説明

### Layer 1: ユーザー側
あなたが使うアプリです。Telegram・WhatsApp・Discord など。

### Layer 2: Bot API
各チャットサービスが提供する公式API。
- Telegram は `api.telegram.org` にWebhookを設定
- メッセージが来たら nanoclaw に転送する

### Layer 3: nanoclaw サーバー ← ここが核心！

| コンポーネント | 役割 |
|--------------|------|
| **メッセージルーター** | どのグループのメッセージかを判断して振り分ける |
| **スケジューラー** | 定期タスクの管理・実行 |
| **グループマネージャ** | 登録グループの設定・管理 |
| **Claude API クライアント** | Anthropic API を呼び出す |

### Layer 4: Anthropic API
Claude が実際に動いているサーバー。
nanoclaw からリクエストを受け取り、応答を生成して返す。

---

## データの流れ（簡略版）

```
1. ユーザーがメッセージ送信
      ↓
2. Telegram Bot API が受信
      ↓
3. nanoclaw の Webhook エンドポイントに転送
      ↓
4. メッセージルーターがグループを識別
      ↓
5. 会話履歴を付けて Anthropic API に送信
      ↓
6. Claude が応答を生成
      ↓
7. nanoclaw が応答を受け取る
      ↓
8. Telegram Bot API 経由でユーザーに送信
```
