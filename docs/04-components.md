# 04. コンポーネント解説 🔧

## nanoclaw の主要コンポーネント

---

## 1. グループ管理 (Group Manager)

グループとは「nanoclaw が応答する登録済みチャット」のことです。

### グループの設定項目

```
グループ設定例:
{
  "jid": "tg:-1001234567890",   ← チャットの一意ID
  "name": "開発チーム",          ← 表示名
  "folder": "telegram_dev-team", ← フォルダ名 (チャンネル_グループ名)
  "trigger": "@Claude"           ← 反応するトリガーワード
}
```

### フォルダ名の命名規則

```
{チャンネル}_{グループ名}

例:
  telegram_family-chat    ← Telegram の "family-chat" グループ
  whatsapp_dev-team       ← WhatsApp の "dev-team" グループ
  discord_general         ← Discord の "general" チャンネル
```

## 2. スケジューラー (Scheduler)

定期的なタスクを管理・実行するコンポーネントです。

### スケジュールタスクの種類

| タイプ | 説明 | 例 |
|--------|------|-----|
| `cron` | 特定の時刻に繰り返し実行 | `0 9 * * *` = 毎朝9時 |
| `interval` | 一定間隔で繰り返し実行 | `3600000` = 1時間ごと |
| `once` | 指定時刻に1回だけ実行 | `2026-03-15T10:00:00` |

### cron 式の読み方

```
┌──── 分 (0-59)
│  ┌─── 時 (0-23)
│  │  ┌── 日 (1-31)
│  │  │  ┌─ 月 (1-12)
│  │  │  │  ┌ 曜日 (0-7, 0と7が日曜)
│  │  │  │  │
*  *  *  *  *

例:
0 9 * * *     → 毎日 9:00
*/5 * * * *   → 5分ごと
0 9 * * 1     → 毎週月曜 9:00
0 9,18 * * *  → 毎日 9:00 と 18:00
```

### コンテキストモード

| モード | 説明 | 使いどころ |
|--------|------|------------|
| `group` | グループの会話履歴を持つ | 「さっきの話の続き」が必要な場合 |
| `isolated` | 毎回新鮮なセッション | 独立したタスク（天気確認など） |

## 3. Claude API クライアント

Anthropic の API を呼び出す部分です。

### 利用できる Claude のツール（Claude Code として）

nanoclaw は Claude に多くのツールを提供しています：

```
ファイル操作系:
  📖 Read        → ファイルを読む
  ✏️  Edit        → ファイルを編集する
  📝 Write       → ファイルを書く
  🔍 Glob        → ファイルを検索する
  🔎 Grep        → ファイル内容を検索する

実行系:
  💻 Bash        → シェルコマンドを実行する

AI エージェント系:
  🤖 Agent       → サブエージェントを起動する

Web系:
  🌐 WebFetch    → URLの内容を取得する
  🔍 WebSearch   → Web検索する

タスク管理系:
  📋 TodoWrite   → TODOリストを管理する

nanoclaw 独自ツール:
  📨 send_message     → ユーザーにメッセージを送る
  ⏰ schedule_task    → タスクをスケジュール登録する
  📋 list_tasks       → タスク一覧を確認する
  ⏸️  pause_task       → タスクを一時停止する
  ▶️  resume_task      → タスクを再開する
  ❌ cancel_task      → タスクをキャンセルする
  ✏️  update_task      → タスクを更新する
  📌 register_group   → グループを登録する
```

## 4. マルチエージェント（チーム機能）

nanoclaw は複数の AI エージェントを同時に動かす「チーム機能」を持っています。

```
チームリーダー (Claude)
    ├── エージェントA: 調査担当
    ├── エージェントB: コード実装担当
    └── エージェントC: テスト担当
```

### チームの仕組み

```
1. TeamCreate でチームを作成
      ↓
2. Agent ツールでサブエージェントを起動
      ↓
3. TaskCreate でタスクを作成・割り当て
      ↓
4. 各エージェントが並行して作業
      ↓
5. SendMessage で進捗報告・コミュニケーション
      ↓
6. 全タスク完了 → TeamDelete でクリーンアップ
```

## 5. Docker コンテナ

nanoclaw はコンテナ上で動いており、必要に応じて**新しいコンテナを起動**できます。

```
nanoclaw が何かを作るとき:
  ↓
Docker コンテナ起動（隔離された環境）
  ↓
その中でコード実行・ファイル操作
  ↓
結果を返してコンテナを破棄（または保持）
```

これにより：
- 🔒 安全な実行環境
- 🧹 後片付けが簡単
- 🔄 再現性の高い実行
