## 演習1: スケジュールタスクを作ってみる

### 目標
「毎朝9時に今日の日付を教えてくれる」タスクを作る

### 手順

Telegram で以下のように話しかけてみてください：

```
「毎朝9時に今日の日付と曜日を教えてください」
```

nanoclaw（Claude）が `schedule_task` ツールを使って登録してくれます。

### 裏で何が起きているか

```
あなた: 「毎朝9時に今日の日付と曜日を教えて」
  ↓
Claude が判断: "スケジュールタスクを作ろう"
  ↓
schedule_task ツールを呼び出す:
  {
    "prompt": "今日の日付と曜日を教えてください",
    "schedule_type": "cron",
    "schedule_value": "0 9 * * *"
  }
  ↓
nanoclaw がタスクを登録
  ↓
毎朝9時に自動実行 ✅
```

## 演習5: グループを新規登録する（上級）

新しいチャットを nanoclaw に登録する場合:

### 必要な情報

| 項目 | 説明 | 例 |
|------|------|-----|
| `jid` | チャットの一意ID | `tg:-1001234567890` |
| `name` | 表示名 | `新プロジェクト` |
| `folder` | フォルダ名 | `telegram_new-project` |
| `trigger` | トリガーワード | `@Claude` |

### Telegram の JID の調べ方

```
1. @userinfobot に "グループに招待" する
2. グループで何かメッセージを送る
3. ボットがグループIDを教えてくれる

例: Chat ID: -1001234567890
JID: tg:-1001234567890
```

## 演習6: このリポジトリの場所を理解する

このリポジトリ自体が「ファイルを作ってもらった」実例です！

```
Docker コンテナ内の場所:
  /workspace/group/nanoclaw-learning/

なぜここか？
  /workspace/project/ → 読み取り専用（ソースコードのマウント）
  /workspace/group/   → 書き込み可能・永続領域 ← ここ！

GitHub に push するには:
  1. GitHub でリポジトリを新規作成
  2. git remote add origin https://github.com/xxx/nanoclaw-learning
  3. git push -u origin main
```
